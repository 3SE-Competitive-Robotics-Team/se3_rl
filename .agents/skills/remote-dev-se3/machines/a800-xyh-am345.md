当前台阶训练优先使用本地脚本托管同步和启动：

```powershell
ssh laptop-imgpi2nm-shanghai "powershell -NoProfile -Command `"cd E:\se3_rl_train; uv run python scripts\remote_sync_start_stair.py --entry-host a800 --pod abbtask-79cdb78487-mgx44`""
```

开发机不能直连 A800。日常流程是开发机先 `git push`，再 SSH 控制 laptop 在 repo 工作区 `git pull --ff-only`，最后从 laptop 运行脚本进入 `a800` 上的 `gczx-project06/abbtask-*` 当前 pod。需要临时切换环境数、任务名或 checkpoint 时，优先通过脚本参数覆盖，不要手改本文档里的默认值。

`scripts/remote_sync_start_stair.py` 默认不同步 `assets/base_model`，避免每次多传约 50 MiB 以上的基模数据。只有需要更新或补齐远端基模 checkpoint 时，显式加 `--sync-base-model`。

## 源码同步

源码同步必须走 git。开发机只负责提交并 push；laptop 拉取同一 commit 后，再从 laptop 同步/启动 A800。

```powershell
$branch = "codex/stair-visualbase-obs34"
$laptopRepo = "E:\se3_rl_train"
ssh laptop-imgpi2nm-shanghai "powershell -NoProfile -Command `"cd '$laptopRepo'; git fetch origin $branch; git checkout $branch; git pull --ff-only origin $branch; git rev-parse --short HEAD`""
```

A800 pod 如果访问 GitHub 报 TLS 错误：

```text
gnutls_handshake() failed: Error in the pull function
```

使用 git bundle fallback：

```powershell
$bundle = "se3_$(git rev-parse --short HEAD).bundle"
git bundle create $bundle HEAD
scp $bundle laptop-imgpi2nm-shanghai:/tmp/$bundle
ssh laptop-imgpi2nm-shanghai "scp /tmp/$bundle a800:/tmp/$bundle; ssh a800 `"kubectl cp /tmp/$bundle gczx-project06/abbtask-79cdb78487-mgx44:/tmp/$bundle -n gczx-project06`""
```

pod 内执行：

```bash
cd /workspace/3SE-Competitive-Robotics-Team/se3_wheel_leg
git fetch /tmp/se3_<commit>.bundle HEAD
git switch <branch>
git merge --ff-only FETCH_HEAD
git rev-parse --short HEAD
```

如果 `merge --ff-only` 失败，先查 `git status` 和最近提交，不要用 rsync 覆盖源码。

## CUDA Forward Compatibility

A800 容器必须让 compat `libcuda.so` 优先生效。训练 shell 中不要把 `/usr/local/cuda-*/lib64` 或 `/usr/local/nvidia/lib64` 提前放进 `LD_LIBRARY_PATH`。启动训练前确认：

```bash
unset LD_LIBRARY_PATH
ldconfig -p | grep -m1 'libcuda.so.1'
```

训练日志必须显示 `Driver 12.6`，且不应出现：

```text
CUDA Graphs disabled: driver 12.2 < 12.4
```

如果看到 `Driver 12.2`，立即停止训练并修正库路径。

## 台阶值守

A800/abbtask 不运行 MJLab Viser。当前台阶远程训练按父目录 `SKILL.md` 的“Viser 值守”和 `docs/laptop_viser_play.md` 值守：

- A800 只负责训练。
- 默认用 GitHub Release checkpoint exchange + 本机 `scripts\local_stair_viser_watcher.py --source github-release`。
- 本机 native MuJoCo closedchain viewer 必须包含 `--viewer viser --stair-terrain --stair-terrain-level <level>`。
- 旧的 Windows laptop native viewer + 开发机转发 laptop 8080 只作 fallback。
- 验收时必须确认台阶是真实 MuJoCo worldbody box 碰撞地形，不是 Viser overlay。

## 产物拉取

checkpoint 默认走 GitHub Release assets 中转：A800/abbtask 只通过 `kubectl cp` 把文件拉到 A800 host，laptop 用内网 `scp` 拉到 `C:\tmp\<tag>\`，再用 `gh release create/upload` 上传到私有仓库 `3SE-Competitive-Robotics-Team/se3_checkpoint_exchange`；本机从 GitHub 下载。不要把 `.pt` commit 进 git 历史。

laptop 已安装并登录 `gh`；本机如果没有 `gh`，使用 Git credential token + GitHub API + `curl.exe` 下载 Release asset。完整命令见父目录 `SKILL.md` 的“拉取 checkpoint”。

临时产物如 Rerun `.rrd`、summary JSON、日志和 `.done` / `.failed` marker 仍可通过 `kubectl cp` 拉取。脚本同时尝试拉 `.done` 和 `.failed` 时，缺少其中一个产生的 `Cannot stat` warning 通常无害；以主产物存在且 `.done`/`.failed` 至少一个存在为准。
