## 只读检查

检查 pod 和 GPU：

```powershell
ssh target-via-phone "kubectl get pod -n gczx-project06 abbtask -o wide"
ssh target-via-phone "kubectl exec -n gczx-project06 abbtask -- nvidia-smi"
```

检查远端仓库：

```powershell
ssh target-via-phone "kubectl exec -n gczx-project06 abbtask -- bash -lc 'cd /workspace/3SE-Competitive-Robotics-Team/se3_wheel_leg && git status --short --branch'"
```

PowerShell 里不要把复杂 bash 直接塞进双引号字符串。复杂操作走 `scripts/remote_bash.ps1` 或当前专用脚本。

## 源码同步

源码同步必须走 git。A800 pod 如果访问 GitHub 报 TLS 错误：

```text
gnutls_handshake() failed: Error in the pull function
```

使用 git bundle fallback：

```powershell
$bundle = "se3_$(git rev-parse --short HEAD).bundle"
git bundle create $bundle HEAD
scp $bundle target-via-phone:/tmp/$bundle
ssh target-via-phone "kubectl cp /tmp/$bundle gczx-project06/abbtask:/tmp/$bundle -n gczx-project06"
```

pod 内执行：

```bash
cd /workspace/3SE-Competitive-Robotics-Team/se3_wheel_leg
git fetch /tmp/se3_<commit>.bundle HEAD
git switch <branch>
git merge --ff-only FETCH_HEAD
git rev-parse --short HEAD
```

如果 `merge --ff-only` 失败，先查 `git status` 和最近提交，不要用 rsync 覆盖源码。

## CUDA Forward Compatibility

A800 容器必须让 compat `libcuda.so` 优先生效。训练 shell 中不要把 `/usr/local/cuda-*/lib64` 或 `/usr/local/nvidia/lib64` 提前放进 `LD_LIBRARY_PATH`。启动训练前确认：

```bash
unset LD_LIBRARY_PATH
ldconfig -p | grep -m1 'libcuda.so.1'
```

训练日志必须显示 `Driver 12.6`，且不应出现：

```text
CUDA Graphs disabled: driver 12.2 < 12.4
```

如果看到 `Driver 12.2`，立即停止训练并修正库路径。

## 台阶值守

A800/abbtask 不运行 MJLab Viser。当前台阶远程训练按 `docs/laptop_viser_play.md` 值守：

- A800 只负责训练。
- Windows laptop 运行 native MuJoCo closedchain `se3-sim2sim --viewer viser --stair-terrain`。
- 开发机只转发 laptop 的 8080 端口。
- 验收时必须确认台阶是真实 MuJoCo 碰撞地形。

## 产物拉取

checkpoint、Rerun `.rrd`、summary JSON、日志和 `.done` / `.failed` marker 可通过 `kubectl cp` 拉取。脚本同时尝试拉 `.done` 和 `.failed` 时，缺少其中一个产生的 `Cannot stat` warning 通常无害；以主产物存在且 `.done`/`.failed` 至少一个存在为准。
