## 必须由用户指定

| 项目 | 说明 |
|---|---|
| Pod 名 | 不在文档中写死，用 `<pod>` 表示 |
| 远端项目路径 | 常见为 `/workspace/3SE-Competitive-Robotics-Team/se3_wheel_leg` 或 `/root/RLleg`，以用户指定为准 |
| CUDA lib64 目录 | 强制使用 `/usr/local/cuda-12.6/lib64`；容器默认 CUDA 运行库太老，不依赖默认 `LD_LIBRARY_PATH` |
| 训练任务和 run 名 | 每次按当前目标指定 |

用户未指定 pod 时先查询：

```powershell
ssh target-via-phone "kubectl get pods -n gczx-project06 -o wide"
```

进入容器：

```powershell
ssh target-via-phone "kubectl exec -it -n gczx-project06 <pod> -- bash"
```

复杂命令使用 base64 传脚本，避免 PowerShell、SSH、kubectl、bash 多层转义问题。

## 训练前环境

必须手动指定 CUDA 12.6 的 `lib64`。这是因为容器默认 `LD_LIBRARY_PATH` 里的 CUDA 运行库版本太老，Python、MuJoCo、Warp 或 Torch 可能优先加载旧库，导致训练运行时异常或 CUDA Graphs 失效。

```bash
cd <remote_project>
export CUDA_LIB_DIR=/usr/local/cuda-12.6/lib64
test -d "$CUDA_LIB_DIR"
export LD_LIBRARY_PATH="$CUDA_LIB_DIR"
unset CUDA_VISIBLE_DEVICES
export MUJOCO_GL=egl
```

如果该目录不存在，先停止并让用户确认容器环境。可在容器里查候选 CUDA 库目录：

```bash
ls -d /usr/local/cuda*/lib64 2>/dev/null
```

需要离线 wandb 时显式：

```bash
export WANDB_MODE=offline
```
