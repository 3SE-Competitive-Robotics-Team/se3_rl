# SerialLeg 机器人背景规格

机器人物理规格、电机参数和速度能力，供训练指令范围、奖励设计和 sim2sim 验证参考。

## 机体结构

- policy 动作自由度：6 DOF（`[LF, LB, RF, RB, l_wheel, r_wheel]`）
- 闭链模型关节语义：`LF/RF = lf0_Joint/rf0_Joint`，`LB/RB = l_drive_bar_Joint/r_drive_bar_Joint`，`lf1_Joint/rf1_Joint` 是被动输出小腿角；主动杆限位语义为同侧两主动杆夹角，当前装配分支下左腿为 `LF-LB`，右腿为 `RB-RF`
- 轮径：**R = 59 mm**（直径 118 mm，来自 MJCF `wheelRadius: 0.059 m`）
- 腿部传动：四连杆机构，膝关节通过驱动杆 AB → 连杆 BC → 小腿 CD 传动
- 轮子安装：小腿末端，接地点固定作为参考坐标系原点

## 轮子最大线速度

基于 M3508-C620-14:1 输出轴转速 × 轮径：

| 工况 | 输出轴转速 | 线速度 |
|------|-----------|--------|
| 空载理论上限 | 68.5 rad/s | **4.04 m/s** |
| 额定扭矩（官方 3 N·m 点按 14:1 换算） | 66.7 rad/s | **3.93 m/s** |
| RL 训练 action 上限（保守低于空载速度） | 45.0 rad/s | **2.66 m/s** |

14:1 输出轴参数由官方 P19 数据按减速比换算：

```
ω_no_load = 482 rpm × 19 / 14 ≈ 654 rpm ≈ 68.5 rad/s
ω_cont    = 469 rpm × 19 / 14 ≈ 636 rpm ≈ 66.7 rad/s
τ_cont    = 3.0 N·m × 14 / 19 ≈ 2.21 N·m
```

为了复用 MJLab 的线性 T-N 包络，代码中的 `stall_torque` 使用经过官方 3 N·m @ 469 rpm 点的外推截距；真实连续输出力矩仍由 `rated_torque=2.21 N·m` 限幅。RL 训练和 sim2sim 的轮子 action scale 当前共享为 `45.0 rad/s`，保守低于 `M3508_C620_14.no_load_speed`，避免把满幅动作直接顶到空载速度边界。`SE3-WheelLegged-Recovery-GRU` 当前轮子速度阻尼 `wheel_kd=0.08`，sim2sim 也使用同一共享默认。

## 动作空间和关节契约

训练默认使用解析四连杆等效开树模型，sim2sim 默认使用真实闭链 OBB 模型；policy contract 仍保持主动杆语义：

```text
[LF, LB, RF, RB, l_wheel, r_wheel]
```

腿部四维是主动电机坐标，其中 `LB/RB` 是独立挂在 `base_link` 下的驱动杆，不是 `lf1/rf1`。`lf1_Joint/rf1_Joint` 仍然存在，但只表示输出小腿角，用于几何诊断、终止、奖励或 critic 特权信息；actor 腿部观测不读取它们。

## 物理形态约束与奖励设计决策

本节记录由机器人物理形态决定的设计约束，以及基于这些约束做出的奖励/终止函数选择。每条决策都有明确的物理依据，不可随意更改。

### 约束 1：接地点只能是轮子，不能是腿

**物理依据**

SerialLeg 是轮腿机器人，腿部结构（lf0_Link、lf1_Link 及对应右侧）是纯传动连杆，没有缓冲结构，也不具备承重设计。腿部 body 触地意味着：
- 结构性异常姿态（机器人已经跌倒或严重侧倾）
- 极高的关节冲击载荷，可能损坏实体机械
- 在仿真中等效于模型失效状态

无论是行走还是跳跃，任何正常运动模式下腿部都不应接触地面。

**设计决策**

`leg_contact` 终止条件（`terminations.py:49`）检测 `lf0_Link / lf1_Link / rf0_Link / rf1_Link` 与地面的接触，一旦发生立即终止 episode。这一终止对所有任务（行走、跳跃）均有效，**没有例外**。

跳跃着陆时，正确的接触顺序是：
1. 轮子先着地（`wheel_sensor` 检测到接触力）
2. 腿部通过 PD 控制吸收冲击，关节角度向默认姿态回弹
3. 全程腿部 body 不接触地面

如果着陆时触发 `leg_contact`，说明：
- 空中姿态失控（tilt 过大），着陆时机身已倾斜到腿部先着地
- 着陆冲击过大导致腿部被压到地面（PD 增益不足以抵抗冲击）
- 这两种情况都是策略需要学会避免的，终止是正确的惩罚信号

**对跳跃训练的启示**

如果训练中 `leg_contact` 终止率偏高（超过 `jump_flag_ratio`），优先检查：
1. RSI 注入的高速姿态是否导致空中 tilt 过大
2. `diag_tilt_airborne_deg` 是否持续超过 15°
3. 域随机化参数（质量、PD 增益）是否导致极端参数组合下无法维持直立

不要为了降低 `leg_contact` 率而放宽终止条件——应该修复导致腿部触地的根因。

### 约束 2：弹簧建模与跳跃能量预算

**物理依据**

膝关节安装气弹簧（MJCF `stiffness=2000 N/m`，`damping=5 N·s/m`），弹簧挂点在驱动杆和小腿上。弹簧储能公式：

```
E_spring = 0.5 × k × Δx²
Δx = r_arm × Δq（杠杆臂 r_arm ≈ 0.04m，压缩量 Δq = q_current - q_default）
```

从默认姿态（lf1=0.207）到 RSI 收腿姿态（lf1=0.4），双侧弹性储能约 0.27 J，仅占 0.8m 跳跃所需 99 J 的 0.3%。**弹簧对跳跃高度贡献可忽略不计，不是调优重点。**

跳跃能量主要来源是腿部关节做功（DM8009P stall_torque=40 N·m，全行程可用功约 104 J）。

---

### 约束 3：PD 控制与 torque_limits 的联动

**物理依据**

训练使用 `DcMotorActuatorCfg`，力矩输出 = `kp × (q_target - q) - kd × dq`，上限由 `torque_limits` 截断。

当 `torque_limits` 从 `rated_torque(20 N·m)` 提升到 `stall_torque(40 N·m)` 后：
- kp 偏低（如 0.8×）时，大误差才能产生大力矩，位置控制精度下降，关节容易漂移
- kp 偏高（如 1.2×）时，力矩饱和频繁，等效 bang-bang 控制，关节加速度高，可能引起振荡和腿部触地

**设计决策**

`torque_limits` 提升后，必须同步收窄 `randomize_pd_gains` 的范围：
- 当前：`kp_range=(0.9, 1.1)`，`kd_range=(0.9, 1.1)`（已收窄）
- 若未来再提升 `torque_limits`，需再次评估是否需要进一步收窄

`leg_dof_acc`（关节加速度惩罚，`src/se3_train/tasks/flat/env_cfg.py`）是监测这个问题的代理指标：若训练中 `Episode_Reward/leg_dof_acc` 绝对值持续偏大（>0.2），说明 PD 振荡严重，应检查 `torque_limits` 与 `kp_range` 的匹配。

### 约束 4：SerialLeg 正确起跳物理流程（最重要的设计依据）

**核心原则：机身质心速度优先，腿部动作服从时序。**

SerialLeg 是轮腿机器人，机身质量远大于腿部（base_link ~10.7kg，腿部合计 ~2kg）。起跳的能量来源是腿部伸展做的功，但决定跳跃高度的是**机身质心的向上初速度**，腿部的收放只是在此基础上通过内力调节。

**完整五阶段时序：**

```
阶段 0 — 蹲下蓄力（grounded, vz ≤ 0）
  膝关节屈曲，气弹簧压缩储能，机身质心降低
  目的：增大腿部伸展行程，蓄积弹性势能

阶段 1 — 蹬地加速（grounded, vz 从 0 开始上升）
  腿部快速伸展，气弹簧释放 + 电机做功
  地面反力推动机身向上加速
  ⚠️ 关键：此阶段腿必须保持充分伸展，不能过早收腿
  过早收腿 = 地面反力作用时间缩短 = 机身获得的冲量减小

阶段 2 — 离地后收腿（airborne, vz 足够大后再收）
  机身已有足够 vz 之后，腿才开始快速向上折叠
  通过内力（腿部质量上移）进一步拉高机身质心
  ⚠️ 关键：不是离地就立刻收腿，而是等 vz 稳定后再收
  收腿过早 = 机身速度还不够就消耗了腿部动量

阶段 3 — 顶点（airborne, vz → 0）
  腿完全收起，整体质心最高点
  姿态保持水平（pitch/roll ≈ 0，yaw 不变）

阶段 4 — 下降展腿（airborne, vz < 0）
  腿开始展开：腿部重心下移，机身相对上移
  等效于减缓机身下降速度，延长机身在高位的时间
  气弹簧随腿展开而伸长，准备着陆缓冲

阶段 5 — 着陆缓冲（landing）
  轮子先着地（约束 1），腿部屈曲吸收冲击
  动量相等时，增大做功时间 = 减小冲力（F = Δp/Δt）
  左右对称，防止偏载翻倒
```

**当前奖励设计的已知问题（2026-05-16 确认）：**

1. **阶段 1 缺乏约束**：早期 `jump_squat_drive` 会奖励蹲下（vz≤0 屈曲），容易把策略推向“主动蹲住”的局部最优；当前已移除该奖励，保留 `jump_takeoff_drive` 等起跳结果信号。仍需关注蹬地时腿是否伸展到底，否则策略可以在 vz 刚变正就立刻收腿，导致地面反力作用时间缩短。

2. **阶段 2 收腿时机过早**：`jump_knee_phase` 在 stage=1（airborne）且 vz≥0 时就立刻奖励收腿。但此时机身 vz 可能还很小（刚离地），过早收腿会把腿部动量低效消耗，不如等 vz 达到阈值后再收。

3. **直接后果（sim2sim 验证）**：`wheel_clr` 峰值只有 0.077-0.09m，大部分"高度"来自收腿而非机身真实飞起，机身 vz 贡献极小。

**修复方向：**
- 蹬地阶段（grounded + vz>0）：增加"腿部保持伸展"约束（惩罚此时的膝关节屈曲量）
- 收腿时机：给 `jump_knee_phase` 加 vz 阈值门控，vz 超过阈值后才奖励收腿
- 地面期核心信号应是：`base_link 向上加速度` 或 `地面反力法向分量`，而不是 vz 的瞬时值
